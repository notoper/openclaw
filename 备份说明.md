# 自动备份配置说明

## 📋 备份策略

**备份频率：** 每小时检查一次  
**备份条件：** 只有在有更改时才备份  
**备份位置：** https://github.com/notoper/openclaw

---

## 🚀 手动测试备份

### 方法1：直接运行备份脚本
```bash
cd /root/.openclaw/workspace-researcher
./auto-backup.sh
```

### 方法2：使用git命令（可选）
```bash
cd /root/.openclaw/workspace-researcher

# 检查状态
git status

# 如果有更改，添加并提交
git add .
git commit -m "备份 - $(date '+%Y-%m-%d %H:%M:%S')"
git push
```

---

## ⏰ 配置定时备份

### 选项1：使用OpenClaw cron（推荐）

在OpenClaw中配置cron任务：

```bash
# 使用OpenClaw的cron功能
openclaw cron create --name "workspace-backup" --schedule "0 * * * *" --command "cd /root/.openclaw/workspace-researcher && ./auto-backup.sh"
```

**cron表达式说明：** `0 * * * *` = 每小时整点执行

### 选项2：使用系统cron

编辑系统crontab：
```bash
crontab -e
```

添加以下行：
```bash
0 * * * * cd /root/.openclaw/workspace-researcher && ./auto-backup.sh >> /var/log/workspace-backup.log 2>&1
```

---

## 📝 备份脚本说明

**脚本位置：** `/root/.openclaw/workspace-researcher/auto-backup.sh`

**脚本功能：**
1. 检查git状态
2. 如果没有更改，直接退出
3. 如果有更改：
   - 添加所有更改
   - 生成带时间戳的提交信息
   - 提交更改
   - 推送到GitHub

**提交信息格式：** `自动备份 - YYYY-MM-DD HH:MM:SS`

## 🛟 恢复备份

如果需要恢复到某个时间点的版本：

```bash
cd /root/.openclaw/workspace-researcher

# 查看所有提交
git log --oneline

# 恢复到某个提交（保留历史）
git checkout <commit-id>

# 或者重置到某个提交（丢弃之后的提交）
git reset --hard <commit-id>
git push --force
```

## ⚠️ 注意事项

1. **确保.gitignore正确配置**
   - 已排除.env、tokens、API密钥等敏感文件
   - 确认敏感信息不会被提交

2. **网络连接**
   - 确保能访问GitHub
   - 推送失败时会报错，需要手动处理

3. **仓库权限**
   - 确保PAT有repo权限
   - PAT过期前需要更新

4. **日志记录**
   - 备份脚本会输出到控制台
   - 配置cron时可添加日志输出
